# SWARM — Руководство пользователя

## Введение

SWARM — это система для координации нескольких LLM-агентов (Claude, Codex, Gemini, OpenCode, Qwen), работающих параллельно над одной кодовой базой. Вы как **Лидер** управляете агентами: создаёте задачи, назначаете их и следите за прогрессом.

---

## Быстрый старт

### 1. Установка

```bash
# Из директории SWARM
pip install -e .
```

### 2. Инициализация проекта

Перейдите в папку вашего проекта и выполните:

```bash
cd ваш-проект
swarm init
```

Это создаст:
- `swarm.db` — база данных для задач и агентов
- `SKILL.md` — инструкции для LLM-агентов (в папках `.claude/`, `.codex/`, `.gemini/` и др.)

### 3. Создание задач

```bash
# Простая задача
swarm task add --desc "Реализовать авторизацию" --priority 1

# Задача для конкретной роли
swarm task add --desc "Спроектировать API" --priority 1 --role architect

# Задача с зависимостью (выполнится после задачи #1)
swarm task add --desc "Написать тесты" --priority 2 --depends-on 1
```

**Приоритеты:** 1 (наивысший) — 5 (наименьший)

**Роли:** `architect`, `developer`, `tester`, `devops`

**Типы CLI:** `claude`, `codex`, `gemini`, `opencode`, `qwen`

### 4. Запуск агентов

Откройте новый терминал для каждого агента:

```bash
# Терминал 1: Запустите Claude CLI
claude

# В Claude скажите:
# "Зарегистрируйся через swarm join"
```

Агент выполнит:
```bash
swarm join
# Введёт: тип CLI, имя, роль
```

**⚠ ВАЖНО: Агенты должны использовать параметр `--agent`**

После регистрации агент должен **запомнить своё имя** и использовать его во всех командах:

```bash
# Регистрация
swarm join --cli codex --name worker1 --role developer

# Все последующие команды — с --agent
swarm next --agent worker1
swarm lock файл.py --agent worker1
swarm done --summary "..." --agent worker1
```

Это позволяет запускать несколько агентов одного типа (например, 5 Codex) без конфликтов.

Повторите для каждого агента в отдельном терминале.

### 5. Запуск монитора

В отдельном терминале:

```bash
swarm monitor
```

Вы увидите live-дашборд с 4 панелями:
- **Агенты** — статус каждого агента
- **Задачи** — очередь задач
- **Блокировки** — какие файлы заблокированы
- **Активность** — последние события

### 6. Начало работы

**В каждом терминале агента** скажите ему текстом:

> "Начинай работать. Выполни `swarm next --agent твоё-имя` чтобы получить задачу."

Агент выполнит цикл:
1. Получит задачу (`swarm next --agent имя`)
2. Заблокирует файлы (`swarm lock файл --agent имя`)
3. Выполнит работу
4. Завершит задачу (`swarm done --agent имя`)
5. Возьмёт следующую задачу

**Важно:** Агенты — это LLM в отдельных терминалах. Они не "слушают" базу данных автоматически. Вы должны **вручную** сказать каждому агенту начать работу.

## Команды Лидера

### Управление задачами

```bash
# Создать задачу
swarm task add --desc "Описание" --priority 1 [--role РОЛЬ] [--cli ТИП] [--name ИМЯ] [--depends-on ID]

# Список задач
swarm task list [--status pending|in_progress|done] [--done]

# Принудительно закрыть задачу (если агент завис)
swarm task close <ID> [--reason "Причина"]

# Назначить задачу конкретному агенту
swarm task assign <ID> --agent <имя>

# Примеры
swarm task add --desc "Рефакторинг базы данных" --priority 2 --role developer
swarm task add --desc "Ревью кода" --priority 3 --name alice
swarm task list --status pending
swarm task assign 5 --agent bob     # Назначить задачу #5 агенту bob
swarm task close 5 --reason "Агент завис"
```

### Управление агентами

```bash
# Список агентов
swarm agents

# Удалить неактивных агентов (мёртвые процессы + >30 мин без heartbeat)
swarm agents --cleanup

# Удалить ВСЕХ агентов (полная очистка)
swarm agents --cleanup --force
```

### Мониторинг

```bash
# Live-дашборд (Ctrl+C для выхода)
swarm monitor

# С полным описанием задач
swarm monitor --full

# TUI-монитор со скроллингом (рекомендуется)
swarm tui

# Настроить интервал обновления (по умолчанию 2 сек)
swarm monitor --refresh 5

# Список задач с полным описанием и резюме
swarm task list            # Активные задачи
swarm task list --all      # Все задачи (включая завершённые)

# Журнал событий
swarm logs
swarm logs --limit 100         # Последние 100 событий
swarm logs --agent alice       # События агента alice
swarm logs --task 5            # События задачи #5
```

**Столбцы в мониторе задач:**
| Столбец | Описание |
|---------|----------|
| ID | Номер задачи |
| P | Приоритет (1-5) |
| Статус | pending/in_progress/done/blocked |
| Роль | Для какой роли задача (`--role`) |
| После | Зависимость (`--depends-on`) |
| Назначен | Кому назначена (`--name` или `task assign`) |
| Работает | Кто сейчас выполняет |

### Блокировки

```bash
# Принудительно снять блокировку (если агент завис)
swarm unlock --force --file src/main.py

# Снять все блокировки
swarm unlock --all --force
```

## Рабочий процесс

### Типичная сессия

1. **Подготовка**
   ```bash
   cd my-project
   swarm init
   ```

2. **Создание задач**
   ```bash
   swarm task add --desc "Спроектировать архитектуру" --priority 1 --role architect
   swarm task add --desc "Реализовать UserController" --priority 2 --role developer --depends-on 1
   swarm task add --desc "Написать тесты" --priority 3 --role tester --depends-on 2
   ```

3. **Запуск агентов** (в отдельных терминалах)
   - Терминал 1: Claude как architect
   - Терминал 2: Claude как developer
   - Терминал 3: Claude как tester

4. **Мониторинг**
   ```bash
   swarm monitor
   ```

5. **Добавление задач по ходу работы**
   ```bash
   swarm task add --desc "Исправить баг в API" --priority 1 --role developer
   ```

### Фильтрация задач

| Параметр | Описание | Пример |
|----------|----------|--------|
| `--role` | Только агенты с этой ролью | `--role architect` |
| `--cli` | Только агенты этого типа | `--cli claude` |
| `--name` | Только агент с этим именем | `--name alice` |
| `--depends-on` | Выполнить после задачи | `--depends-on 5` |

### Зависимости задач

```bash
# Задача #1: архитектура
swarm task add --desc "Спроектировать" --priority 1

# Задача #2: реализация (после #1)
swarm task add --desc "Реализовать" --priority 2 --depends-on 1

# Задача #3: тесты (после #2)
swarm task add --desc "Тесты" --priority 3 --depends-on 2
```

Задача #2 не будет доступна, пока #1 не завершена.

## Решение проблем

### Агент завис / не отвечает

1. Проверьте heartbeat в `swarm agents`
2. Если >5 минут — агент вероятно завис
3. **Закройте его задачу принудительно:**
   ```bash
   swarm task close <ID> --reason "Агент завис"
   ```
   Это автоматически снимет блокировки и освободит агента.

4. Или снимите блокировки вручную:
   ```bash
   swarm unlock --force --file заблокированный_файл.py
   swarm unlock --all --force   # Все блокировки
   ```

5. Удалите мёртвых агентов:
   ```bash
   swarm agents --cleanup
   ```

### Задача не назначается

Проверьте фильтры:
- Есть ли агент с нужной ролью?
- Завершена ли зависимость (`depends_on`)?

```bash
swarm task list    # Проверить статус задач
swarm agents       # Проверить роли агентов
```

### Конфликт файлов

Система предотвращает конфликты через блокировки. Если агент ждёт файл:
- Это нормально — он подождёт освобождения
- В мониторе видно, кто блокирует файл

### Очистка

```bash
# Удалить неактивных агентов (по PID и heartbeat)
swarm agents --cleanup

# Удалить ВСЕХ агентов
swarm agents --cleanup --force

# Переинициализировать (осторожно — удалит все данные)
swarm init --force
```

## Лучшие практики

1. **Создавайте все задачи до запуска агентов** — так приоритеты будут корректными

2. **Используйте роли** — architect для проектирования, developer для кода, tester для тестов

3. **Пишите чёткие описания** — агенты это LLM, им нужны ясные инструкции

4. **Следите за блокировками** — долгие блокировки (>30 мин) могут указывать на проблему

5. **Используйте зависимости** — для последовательных задач

6. **Держите монитор открытым** — так вы видите прогресс в реальном времени

## Дашборд монитора

Есть два варианта:
- `swarm monitor` — простой Rich-дашборд
- `swarm tui` — TUI со скроллингом (рекомендуется для большого числа задач)

```
┌─────────────────────────────┬───────────────────────────────────────────────┐
│       ПАНЕЛЬ АГЕНТОВ        │              ПАНЕЛЬ ЗАДАЧ                     │
│                             │                                               │
│  #1 claude/alice/architect  │  ID  P  Статус    Роль  После Назнач. Работает│
│     Статус: WORKING [#7]    │  #7  1  🔄 in_prog dev    -    alice   alice  │
│     Heartbeat: 12с          │  #8  2  ⏳ pending test   #7   bob      -     │
│                             │  #9  3  ⏳ pending  -     #8    -       -     │
│  #2 codex/bob/developer     │                                               │
│     Статус: IDLE            │                                               │
├─────────────────────────────┼───────────────────────────────────────────────┤
│    ПАНЕЛЬ БЛОКИРОВОК        │    ПАНЕЛЬ АКТИВНОСТИ                          │
│                             │                                               │
│  src/api.py -> alice [12м]  │  14:32 ▶ #7 alice  Task started               │
│  src/db.py  -> alice [12м]  │  14:31 ✓ #6 bob    Task done                  │
│                             │  14:30 🔒    alice  Lock api.py               │
└─────────────────────────────┴───────────────────────────────────────────────┘
```

**Панель задач показывает:**
- **Роль** — для какой роли задача (`--role architect`)
- **После** — зависимость, после какой задачи (`--depends-on 5`)
- **Назначен** — кому назначена задача (`--name` или `task assign`)
- **Работает** — кто сейчас выполняет

**Панель активности показывает:**
- Номер задачи (`#7`) для событий, связанных с задачами
- Имя агента, выполнившего действие
- Тип события (▶ старт, ✓ готово, 🔒 блокировка)

**Управление в TUI (`swarm tui`):**
- **↑↓** — скроллинг списков
- **R** — обновить данные
- **D** — показать/скрыть завершённые
- **Q** — выход

**Цвета статусов:**
- 🟢 Зелёный — working (работает)
- 🟡 Жёлтый — waiting (ждёт блокировку)
- ⚪ Серый — idle (ожидает задачу)
- 🔴 Красный — heartbeat >5 мин (возможно завис)

```bash
swarm --help             # Список всех команд

swarm task close --help  # Справка по закрытию задачи
swarm agents --help      # Справка по агентам
```
