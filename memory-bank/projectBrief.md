# SWARM — Краткое описание проекта

## Цель
SWARM — локальная система оркестрации мультиагентной среды для координации нескольких LLM-агентов (Claude, Codex, Gemini и др.), работающих параллельно над общей кодовой базой.

## Ключевые задачи
1. **Единый CLI-интерфейс** (`swarm`) для управления мультиагентными рабочими процессами
2. **Распределение задач** — создание, приоритизация и назначение задач агентам
3. **Предотвращение конфликтов** — блокировка файлов при параллельном редактировании
4. **Мониторинг в реальном времени** — дашборд с состоянием агентов и задач
5. **Полностью локальная работа** — без облачных зависимостей

## Не-цели (v1)
- Автономное общение агентов между собой
- Облачная очередь задач или брокер сообщений
- Веб-интерфейс (только терминал)
- Автоматическое код-ревью или интеграция с CI/CD

## Ключевые принципы
- **Ручное управление**: Лидер явно управляет агентами
- **Постоянство сессии**: Терминалы агентов не закрываются после задач
- **Детерминированное назначение**: Строгая цепочка фильтров (роль → имя → тип CLI)
- **Безопасность файлов**: Атомарная блокировка файлов

## Целевая платформа
Windows 11, PowerShell, локальная машина
